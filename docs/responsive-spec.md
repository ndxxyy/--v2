# 小钟岐医 V2 响应式规范

## 1. 适用范围

本文冻结“青瓷编目”在桌面、平板与手机上的栅格、信息顺序、图像行为和交互边界。它服务任务 03–08；视觉令牌以 `DESIGN.md` 和 `src/styles/tokens.css` 为准。

响应式不是桌面版等比缩小。窄屏时先保留身份、真实作品、当前内容和主要动作，再移动辅助说明、元数据和次要入口。

## 2. 验收视口与支持下限

| 场景 | 视口 | 用途 |
| --- | --- | --- |
| 最小支持宽度 | 320px | 确认无横向滚动、控件不裁切 |
| 标准手机 | 390×844 | 任务 03–08 的手机截图与交互验收 |
| 大手机/小平板 | 480–767px | 检查筛选换行和长文本 |
| 平板竖屏 | 768×1024 | 8 栏布局、导航切换边界 |
| 平板横屏/小桌面 | 1024×768 | 双栏比例、弹窗高度 |
| 标准桌面 | 1440×900 | 任务 03–08 的桌面截图与首屏验收 |
| 宽桌面 | 1600px 以上 | 确认内容最大宽度和外部留白 |

页面最小宽度为 320px。任何业务页面不得依赖横向滚动；只有大图浏览器中的受控图像平移可以例外，且不得带动整个页面。

## 3. 断点与栅格

| 范围 | 栏数 | 页面边距 | 栏沟 | 说明 |
| --- | ---: | ---: | ---: | --- |
| `< 768px` | 4 | 18px | 16px | 单列内容为默认，局部可 2 栏 |
| `768–1279px` | 8 | 24–32px | 20px | 双栏或主/辅信息布局 |
| `≥ 1280px` | 12 | 48px | 24px | 最大内容宽度 1280px |

- 使用 CSS Grid 时，每一栏必须 `minmax(0, 1fr)`，避免长词撑破容器。
- 正文最大 72ch；弹窗与说明文字不得因宽屏无限增长。
- 区块垂直间距使用 `clamp(64px, 8vw, 128px)`，同一信息组使用 8–24px。
- 视觉参考中的品牌侧栏仅作为任务 03 可选的桌面框架；小于 1280px 时必须并入顶部 Header，不保留压缩后的窄侧栏。

## 4. 全局框架

### 4.1 Header

桌面：

1. 品牌标志或文字标识。
2. 主导航：首页、图谱库、医案整理、授权合作、关于作者、联系与交流。
3. 语言入口。

导航可以是顶部单行，也可以在 `≥1280px` 使用品牌侧栏 + 顶部导航。当前项必须有 `aria-current="page"` 和可见下划线。内容在 200% 缩放时允许换行，不裁切导航文字。

手机：

1. 品牌标识。
2. 当前语言。
3. 菜单按钮。

展开菜单后先列内容入口，再列信任/联系入口，最后列语言。菜单按钮与关闭按钮最小 44×44px；打开后焦点进入菜单，Tab 不逃出，ESC 关闭，关闭后焦点返回菜单按钮。背景页面不可滚动。

### 4.2 Footer

桌面按三组排列：图谱与内容、信任与授权、联系与语言。手机改为纵向顺序，组标题与链接左对齐，组间使用 1px 分隔线。

只显示真实可用项目。缺失版权号、机构、地址、社媒或联系方式时整项隐藏；不得保留空列、破折号或“建设中”。

## 5. 页面信息顺序

### 5.1 首页

桌面顺序：

1. 品牌首屏：品牌名、价值说明、主要动作和至少一件真实作品同屏。
2. 图谱精选。
3. 医案精选或合规空状态。
4. 专业方向。
5. 授权合作。
6. 联系与交流。

手机顺序：

1. 紧凑品牌身份与一句价值说明。
2. 首件真实精选作品及必要标题/分类。
3. 浏览图谱的主要动作与其余图谱精选。
4. 专业方向。
5. 医案精选或合规空状态。
6. 授权合作。
7. 联系与交流。

手机首屏不得被统计数字、长简介或装饰图占满。若尚无真实作品数据，任务 03 不得用占位作品代替；应等待任务 04 的公共选择器或使用明确但克制的内容状态，且不宣称已有作品上线。

### 5.2 图谱目录与分类页

桌面：标题/说明与计数 → 搜索/筛选 → 当前条件 → 作品目录。筛选可占 3–4 栏，作品占其余栏；也可使用顶部筛选带，但第一屏必须看见真实缩略图。

手机：标题与数量 → 搜索 → 可换行筛选 → 当前条件/清除 → 作品列表。筛选不得用被裁切的横向滚动条隐藏关键选项。作品默认单列，只有图像与标题长度验证通过时才允许双列。

### 5.3 图谱详情与大图

桌面：作品图像为 7–8 栏，作品标题、编号、分类、说明与动作占 4–5 栏；缩略图导航在主图下或侧边，不能挤压主图。

手机严格按以下顺序：

1. 返回图谱目录。
2. 作品标题、分类与真实编号（存在时）。
3. 简短说明与授权/联系动作。
4. 主图。
5. 图组缩略图与图片说明。
6. 版权和更多作品（仅真实存在时）。

大图弹窗在手机上近全屏，但保留 18px 安全边距；关闭按钮固定在可见区域。支持上一张、下一张、方向键和 ESC。打开时焦点进入弹窗，关闭后回到原缩略图或“查看大图”按钮。

### 5.4 医案列表与详情

列表在所有宽度保持文献索引语法，不转为图片卡片。手机顺序为分类/日期 → 标题 → 摘要 → 阅读动作。没有公开医案时只显示栏目用途、内容审核边界与相关克制入口，不显示假文章标题或空列表计数。

详情手机顺序为标题与审核过的公开元数据 → 内容 → 医学边界提示 → 了解作者/就诊说明/更多医案。正文图片必须按内容模型提供固定宽高，禁止原始检查单、患者面部或可识别资料。

### 5.5 授权、关于与联系

授权页手机顺序为使用场景 → 边界/材料 → 获取报价动作；无确定价格时只写“获取报价”。从图谱进入时，作品编号紧邻页面标题显示，不能埋在表单中。

联系页始终先区分“就诊咨询”和“学习交流”，再显示各自真实渠道。手机上每个电话、邮件、复制或外链动作占独立行并满足 44px 触控区。不创建不可提交表单。

关于页先显示经审核的作者简介，再显示有证据的资质；资质为空时整个区域隐藏，不使用默认头衔。

## 6. 组件响应式行为

### Buttons and links

- 手机主要按钮可占满容器，但短动作不强制满宽；同组按钮在 `<480px` 时纵向排列。
- 按钮文字允许自然换行但不得被截断；动作名称优先保持 2–8 个汉字。
- 正文链接保持下划线，触控链接周围需有足够行高或内边距。

### Filters

- 默认换行，不使用胶囊式无限横向滑轨。
- 条件多于 6 个或包含多组字段时，手机使用一个可命名的“筛选”按钮打开弹窗，并在按钮旁显示已选数量。
- 结果数量、当前条件和清除动作在 320px 下仍需完整可见。

### Work entries

- 目录缩略图按真实资源选择 `3:4`、`4:3` 或 `1:1`，容器在加载前已有尺寸。
- 桌面图文可并列；手机通常先图后信息。图谱详情是例外，必须先信息后主图。
- 编号、版权、摘要均为可选行；隐藏后布局自动收拢，不保留空白槽位。

### Clinical case summaries

- 桌面元数据可独占 3 栏，正文 9 栏；手机元数据移到标题之前。
- 摘要最多显示经审核的完整短摘要，不用 CSS 截断把医疗语义截成误导句。

### Notices

- 桌面内边距 24px，手机 20px；宽度跟随内容列，不悬浮覆盖作品。
- 多个提示连续出现时合并同类信息，避免警告墙。

### Dialogs

- 宽度为 `min(640px, calc(100vw - 36px))`，最大高度 `calc(100dvh - 36px)`。
- 桌面内边距 32px，手机 20px；标题/关闭按钮固定，内容区独立滚动，操作区置底。
- 320px 宽度下操作按钮纵向排列。弹窗内部不得再嵌套卡片式弹窗。

## 7. 图像与加载

| 用途 | 比例 | 1x 建议尺寸 | 2x 建议尺寸 | 格式 |
| --- | --- | --- | --- | --- |
| 竖版目录缩略图 | 3:4 | 480×640 | 960×1280 | WebP/AVIF |
| 横版目录缩略图 | 4:3 | 640×480 | 1280×960 | WebP/AVIF |
| 方形封面 | 1:1 | 560×560 | 1120×1120 | WebP/AVIF |
| 详情公开预览 | 保持真实比例 | 最长边 1440 | 按任务 04 管线 | WebP/JPG/PNG |
| 分享图 | 1.91:1 | 1200×630 | 不需要 2x | JPG/PNG |

- `sizes` 必须反映实际列宽：手机约 `calc(100vw - 36px)`，桌面按 5/12、6/12 等真实跨栏值填写。
- 首屏真实作品可使用高优先级；首屏外图片懒加载。
- 加载占位为固定比例单色面与轻微透明度脉冲，不用渐变 shimmer；减少动态时取消脉冲。
- 图片失败时显示明确文字与重试/返回动作，不把 `alt` 当作视觉占位布局。

## 8. 文本扩展与三语言

- 简体、繁体和英文共享结构但不共享固定高度。
- 英文导航和按钮预留约 40% 文本扩展空间；不得用减小字号解决溢出。
- 长英文单词使用 `overflow-wrap: anywhere` 仅作为兜底；正常标题优先自然换行。
- 中文正文不使用全大写、超宽字距或强制两端对齐。
- 语言切换后保持当前实体和查询参数；不存在的翻译按任务 07 的策略返回对应目录。

## 9. 交互与无障碍验收

- 全站键盘顺序与视觉顺序一致。
- 所有交互可在无鼠标条件下完成；焦点圈不被 `overflow` 裁切。
- 触控目标至少 44×44px，相邻目标间至少 8px。
- Hover 不能承载手机无法获得的唯一信息。
- `prefers-reduced-motion` 下取消平移、缩放、自动轮播和循环动画。
- 200% 浏览器缩放和 320px 宽度下不丢内容、不叠控件、不出现整页横向滚动。
- 深色品牌面上的文字使用宣纸白或同色相高亮阶，不使用灰字。

## 10. 后续任务截图检查表

在 1440×900 与 390×844 下逐页确认：

1. 第一屏是否出现正确的主信息和真实作品。
2. H1、主要动作、当前导航和语言是否清楚。
3. 图片比例是否稳定，加载前后是否无明显布局位移。
4. 长标题、英文、作品编号和筛选条件是否完整。
5. 是否存在横向滚动、文字溢出、控件重叠或触控区过小。
6. 键盘焦点、ESC、弹窗焦点限制与关闭后焦点返回是否正确。
7. 缺失字段是否真正隐藏，页面是否出现假数据、占位符或无效控件。
