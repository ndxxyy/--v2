## 1. 当前项目目标

建设“小钟岐医个人独立网站 V2”，将真实中医图谱、合规医案内容、授权合作、版权边界、作者介绍及联系入口组织为一个三语言独立网站。

产品与视觉方向为“青瓷编目”：现代中医视觉档案馆、医学出版物与瓷器编目的结合。网站需要同时满足：

- `/zh-CN`、`/zh-Hant`、`/en` 三个独立语言路径。
- 第一屏直接展示真实作品，不以装饰性插画或空洞品牌口号代替内容。
- 图谱目录清晰、快速，并支持搜索、分类、详情和可访问的大图浏览。
- 医案只有完成匿名化、患者授权、编辑审核和发布日期确认后才可公开。
- 就诊咨询与学习交流明确分开，不提供远程诊断、虚假表单或疗效承诺。
- 高清母版不公开，公开站只使用经过审核的缩略图、预览图和分享图。
- 不复刻旧网站的布局、组件和 CSS，只迁移可核验的真实内容与素材。

## 4. 关键技术决策

| 项目 | 决策 |
| --- | --- |
| 框架 | Next.js 16.2.10，App Router，服务器组件优先 |
| UI 运行时 | React / React DOM 19.2.7 |
| 语言 | TypeScript 5.9.3，`strict` |
| 样式 | Tailwind CSS 4.3.2，CSS-first 令牌系统；业务页面大量使用 CSS Modules |
| 包管理 | npm 11，唯一锁文件为 `package-lock.json` |
| Node.js | `>=24 <27`，当前验证环境为 24.15.0 |
| 多语言 | locale key 固定为 `zh-CN | zh-Hant | en`，每种语言独立 URL |
| 内容维护 | 暂不使用数据库或后台，真实内容保存在受版本控制的 TypeScript 数据文件中 |
| 数据流 | `src/data` 审核记录 → 纯函数公开选择器 → 服务器页面 → 最小客户端交互数据 |
| 交互边界 | 搜索、筛选、画廊、复制和移动导航使用客户端组件，其余默认服务器组件 |
| 图片安全 | 母版置于非公开目录；目录只加载 WebP/AVIF 缩略图，详情使用公开预览图 |
| 医案发布 | 默认不公开，四重审核加真实发布日期和安全正文是硬门槛 |
| 缺失资料 | 字段直接隐藏，不展示占位符、假数字、假入口或示例登记号 |
| 授权价格 | 无真实价格时一律使用“获取报价” |
| Impeccable | 项目级安装，自动 Hook 关闭；只在明确阶段运行检查命令 |

4. SEO、canonical、hreflang、sitemap、robots、本地化 404 和结构化数据尚未实现。

6. 正式构建在受限沙箱内可能停在 Turbopack 临时端口阶段；在正常构建权限下已通过。这是执行环境限制，不是已确认的代码错误。
